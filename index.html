What is Vagrant?

Uses type-2 hypervisors (providers) to build and configure portable and complete development environments.

VirtualBox is the default provider

Why Vagrant?

  • No more "works on my machine" excuses
  • Standardize development and production environments
  • Easy setup and configuration
  • Share environments with team members

Important Commands

  • vagrant halt (shutdown)
  • vagrant up (boot)
  • vagrant reload (reboot)
  • vagrant status
  • vagrant destroy (delete box)
  • vagrant suspend (pause box)
  • vagrant ssh
  • vagrant box outdated
  • vagrant box update

What is Scotch Box?

Preconfigured vagrant box with a full array of LAMP stack features.

Why Scotch Box?

  • Quick setup
  • Easy configuration
  • When you need a quick and easy dev environment

Provisioning is your answer.

What are provisioner scripts?

Scripts that can be run for setup and configuration.

To provision:

  • Add --provision flag to an up or reload command.
  • Provision while box is running: vagrant provision

How It Works

Install VirtualBox or another provider (VMWare, Docker, etc)

Install Vagrant

Clone Scotch Box

Run vagrant up

Browse to IP (http://192.168.33.10/ by default)
